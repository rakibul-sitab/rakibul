Data description:

  1. Name
  2. Platform
  3. Year_of_Release
  4. Genre
  5. NA_sales (North American sales in USD million)
  6. EU_sales (sales in Europe in USD million)
  7. JP_sales (sales in Japan in USD million)
  8. Other_sales (sales in other countries in USD million)
  9. Critic_Score (maximum of 100)
  10. User_Score (maximum of 10)
  11. Rating (ESRB) Data for 2016 may be incomplete.

Step 1 - Understanding the data :

In the beginning, the datasets have been opened and the general information have been analysed .The data set has 16715 rows, 11 columns. Most of the missing values are in the scores and rating columns. Also year of release column has some missing values, and two rows with missing values in the names and genres columns have been found.No duplicated row were observed.

Step 2 - Data preprocessing:

  1. All the column names have been converted to lowercase.After that the missing columns have been identified in the games_df dataframe. The missing percentage have been also calculated.Highest missingness is found in the critic_score column and then in the rating and user_score columns.

  2. I used two missing correlation method. One is heatmap and another one is dendrogram. Heatmap shows nullity correlation between various columns and dendrogram shows hierarchical cluster based on missing values correlation between various datasets.We can notice from this heatmap that name and genre has a correlation of 1.0 which means that if the value from one column is missing then the value in another column will be missing as well.The columns of the dataset which have a deep connection in missing values between them will be kept in the same cluster in dendrogram.We can notice that name and genre form one cluster. The other_sales, jp_sales, eu_sales, platform, na_sales creates another cluster. We can notice a good correlation of 0.8 between critic_score-rating and critic_score-user_score which depicts the same missing value relationship.

  3. Using dendogram missingness correlation have been illustrated.Then null values from year_of_release column have been deleted and column type have changed to integer to look without decimals.Then non specific two genre category have been removed which is also correlated with the name column.There is a outliers in the critic_score and user_score column.Null value of columns user_score and critic_score are filled with the median value using trasnform method according to genre and year_of_release columns group value as both contains outliers and rest are also filled with median.Before filling null values in the user_score column, the ‘tbd’ values present in user_score column have replaced with NaN values and the column type have been changed to float.Then all the category are identified in the rating column using value_counts() method and filled with category based on percentage method.

Step 3 - Data Analysis:

  1. Using pivot table method from games_df dataframe games_num_pivot table is created to count the total number of games releases throuhout the data set years.There is no direct relationship found between total games number and release year.But the highest number of games was released in 2008 and it is almost 1427 numbers.But a significant down trend of game production observed after 2008.The last year 2016 , only 502 games were released.

  2. Using pivot table method from games_df dataframe sales_platform_pivot table is created to sum the total amount of revenue earned by the different platforms.The highest revenue earned by PS2,X360,PS3,Wii and DS platforms.

  3. To see the popular platforms sales_revenue distribution over the years, a new greatest_sales_pivot table is created.To illustrated the distribution, plotly library have been used.The average life cycle of popular platforms is about 10 years except DS platform.Normally, It takes 4-5 years to get the sales_revenue peak.Wii, X360 and PS3 platforms were launched in around 2005 But they have almost zero sales now.So to build a prognosis/forecast for 2017, we should look the data around 2012-2013.

  4. The leading platforms are X360,PS3,Wii and DS.But PSP,PS2,Wii and DS are shrinking.The potential platforms are PS4,X360,PSV,3DS,WiiU and XOne.

  5. Total revenue by platforms and total average revenue are slightly different.The all time revenue leaders are PS2, Wii, X360, PS3 and DS.But the average revenue leaders are PS4, XOne, PS3,X360 and Wii.The most promising one X360 and PS4.

  6. PS4 and X360 platforms have been chosen to see the affect on sales by user and professional reviews.There is a positive correlation have been found between critic_score and total_sales for both platforms. Higher critic score brings more sales.Most of the critic score lies between 70-80 for PS4 and 70-90 for X360, and these range can brings 1-2 million USD for PS4 an 4-5 millions usd in pocket by X360 .If it crosses 80, few millions could be earned for PS4 platform.And if crosses score 90, few millions more could be earned.It is clear that higher critic score brings more revenue.

On the other hand, a negative relationship have been observed between user_score and total_score for PS4 and slightly positive more X360.Majority user score lies in the range between 7-9 for both platforms.So, my recommendation is to stick with crictic_score for the prognosis 2017.

  1. To get the distribution of games by genre,potential_genre_pivot table is created.The most profitable genres are Action,Sports and shooter.The lowest profitable genre are strategy and puzzle.

Step 5 - Hypotheses Testing:

  1. games_df_filtered dataframe two seperated dataframes xbox_df and pc_df have been created using query method. The analysis suggested that the average revenue from users of ultimate and surf calling plan doesn’t differs.

  2. games_df_filtered dataframe two seperated dataframes sports_df and action_df have been created using query method. The analysis suggested that Average user rating for the action and sports genres are different.In both cases, a significance level of 0.05 selected which concludes a 5% risk of a difference exists when there is no actual difference. Lower significance levels indicate that i require stronger evidence before i reject the null hypothesis.
