Project 1: Build best performing machine learning models for credit card fraud detection

August 1, 2022

Business problem:

Credit card companies can detect fraudulent credit card transactions, preventing customers from being charged for products they did not purchase. Data Science and Machine Learning can be used to solve this type of problem. This project aims to demonstrate to build a best performing machine learning models to detect Credit Card Fraud.

Dataset:

The Dataset collected via Kaggle which covers credit card transactions made by European cardholders in September 2013. The Dataset contains just 492 fraud cases out of a total of 284807 transactions. The Dataset is unbalanced, with positive transactions (Frauds) accounting for only 0.17% of total transactions. There is no missing data and duplicate data have been removed.

Machine learning model development:

The dataset is highly unbalanced, so the model has been trained in a way that is not overfitted to either of the classes and for this, under sampling methods have been addressed. The test data are not under sampled for the model to work well with highly skewed data. Fraud behaves as an anomaly, so no outlier removal was carried out. if we removed outlier, our model would not have the relevant data to train. Several models have been developed for under-sampling strategies, including logistic regression, random forest, support vector, decision trees, and K-nearest neighbors. The Imbalanced-learn library’s NearMiss for under sampling techniques have been implemented.

AUC represents the degree or measure of separability, while ROC is a probability curve. It indicates how well the model can distinguish between classes. The greater the AUC, the better the model predicts 0 classes as 0 and 1 classes as 1. The ROC curve for under sampling clearly shows that Logistic regression, Support vector, and K-nearest neighbors are the best models, with support vector (AUC: 0.958) being the top.

Accuracy is a metric that generally describes how the model performs across all classes. It is useful when all classes are of equal importance. It is calculated as the ratio between the number of correct predictions to the total number of predictions. The recall is calculated as the ratio between the number of Positive samples correctly classified as Positive to the total number of Positive samples. The recall measures the model’s ability to detect positive samples. The higher the recall, the more positive samples detected. For instance, in fraud detection. If a fraudulent transaction (Actual Positive) is predicted as non-fraudulent (Predicted Negative), the consequence can be very bad for the bank. The support vector achieved the highest accuracy, and the random forest achieved the highest recall in the table above. However, when all measures are combined, logistical regression (Accuracy: 74%, Recall: 87%) and K-nearest neighbor (Accuracy: 81%, Recall: 86%) may be the best models for under-sampling approaches.

Future work: In the future, the oversampling method for the relevant models could be analyzed using SMOTE approaches.
