Project 2: Build multiclass text classification model for product category using AutoML(DataRobot).

August 25, 2022

Business problem:

Customers are the most important for any business. In a large company, thousands of complaints arrive and must be handled properly. Each company has specific specialists who are responsible for particular problems/complaints. When a complaint comes, it must be transferred to the appropriate department. Auto ML could solve this problem and identify product categories from large text complaints.

Dataset :

• Dataset contains more than two million customer complaints about consumer financial products.

• Dataset has been collected from Consumer Complaint Database.

• Approx. 35K rows have been selected for this project.

• Among the 18 columns, only a column contains the actual text of the complaint and one column contains the product for which the customer is raising the complaint has been selected.

Preprocessing text data:

Complain Text raw data was preprocessed in Python using text preprocessing techniques such as tokenizing, removing stop words, and lemmatization.

Product mapping :

Product mapping was performed to improve model accuracy.

Word Embeddings/vectorization with Spacy :

• Machine learning models accept (an array of numbers) as input.

• English pre-trained models from spacy(en_core_web_lg) have been used to get a 300 dimension vector representation of each complain sentence.

Train / Test split data in DataRobot :

DataRobot splits the whole data into third ,For example, 64% for training, 16% for validation, and 20% for holdout.

DataRobot Leaderboard :

Python notebook was connected to DataRobot account via API and asked DataRobot to find the best performing models. For this dataset, DataRobot’s AutoML capabilities advised light gradient boosted trees classifiers with an early stopping model. The cross-validation accuracy was 82%, with an AUC of 0.9617. And the holdout, which is unseen data for the model, had an accuracy of 82% and an AUC of 0.9634.

Model Evaluation:

Confusion metrics were utilized to assess the model. Because the dataset was unbalanced, the F1 score is the best metric for the evaluation. This model performs well in any complaint about a ‘credit card,’ and it has a 0.92 F1 score in this class. Any complaint about the loan class, the model performs badly. For example, the F1 score for the loan class is just 0.45, and the F1 score for the vehicle loan class is only 0.30.

Demo to predict unseen text data with datarobot:

The following complain unseen text for trained model has been tested from Python to DataRobot using API to determine which product category this belongs to. And the outcome indicated that it belongs to the credit report category.

“Equifax is ignoring my requests to investgate, verify and remove accounts from my credit report that do not belong to me.”

Future Recommendation :

Different vectorization trained models could be explored in the future Like Word2Vec, GLOBE or BERT Transformer model.
